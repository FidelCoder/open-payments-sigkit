# Roadmap

## Near Term

- Expand fixed fixtures with more Open Payments request shapes
- Add richer `Signature-Input` diagnostics for malformed structured fields
- Broaden CLI ergonomics around file pipelines and saved outputs
- Add more verification fixtures that mirror real interoperability failures

## Next Stage

- Publish the core package and CLI as versioned open-source artifacts
- Add conformance-oriented fixtures and golden signature-base outputs
- Add JWKS selection diagnostics for multi-key registries
- Document interoperability guidance for generated SDKs

## Future Extensions

- Algorithm abstraction for future signing methods if Open Payments needs broaden
- More detailed inspection views for per-component canonicalization
- Dedicated fixture packs for SDK and test harness generation

The scope should remain focused on signing, verification, debugging, and conformance support for Open Payments HTTP message signatures.
